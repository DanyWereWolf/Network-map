# Пробная версия: как отдать программу «на попробовать» и сохранить авторские права

Ниже — практические шаги, которые помогают зафиксировать ваши права и снизить риски при передаче программы третьим лицам.

## Ограничения использования (пробный период и лимиты)

В конфиге сервера **server-config.json** (скопируйте из **server-config.example.json**) можно включить:

| Параметр     | Описание |
|-------------|----------|
| `trialDays` | Пробный период в днях. При первом обращении к серверу фиксируется дата начала; после истечения редактирование и сохранение блокируются. `0` — без ограничения по времени. |
| `maxObjects` | Максимальное число объектов на карте (узлы, кроссы, муфты, опоры и т.д.). `0` — без лимита. |
| `maxCables` | Максимальное число кабелей. `0` — без лимита. |

Пример для пробной версии на 30 дней и не более 50 объектов и 20 кабелей:

```json
{
  "port": 3000,
  "host": "0.0.0.0",
  "trialDays": 30,
  "maxObjects": 50,
  "maxCables": 20
}
```

При срабатывании ограничения в интерфейсе показывается красный баннер, режим редактирования отключается, добавление объектов и кабелей блокируется. Сохранение на сервере также не выполняется (в т.ч. по WebSocket).

Чтобы снять ограничения, установите в конфиге `trialDays: 0`, `maxObjects: 0`, `maxCables: 0` и перезапустите сервер.

## Ключи подписки (продление на 7 дней, 30 дней, 6 месяцев, 1 год)

Ключи генерирует **только создатель приложения** отдельным скриптом. Приложение только проверяет ключ и сохраняет срок подписки.

### Настройка сервера

В **server-config.json** задайте секрет (тот же, что используется при генерации ключей):

```json
"licenseSecret": "ваш_секретный_строка"
```

Секрет не должен попадать в репозиторий. Можно задать его переменной окружения: `LICENSE_SECRET=ваш_секрет`.

### Генерация ключей (только у создателя)

Скрипт **license-key-gen.js** не привязан к основному приложению; его хранят отдельно и запускают только для выдачи ключей.

**Запуск:**

```bash
LICENSE_SECRET=ваш_секрет node license-key-gen.js <срок>
```

**Сроки:** `7d` | `30d` | `6m` | `1y` (7 дней, 30 дней, 6 месяцев, 1 год).

Примеры:

```bash
LICENSE_SECRET=mySecret123 node license-key-gen.js 7d
LICENSE_SECRET=mySecret123 node license-key-gen.js 30d
LICENSE_SECRET=mySecret123 node license-key-gen.js 6m
LICENSE_SECRET=mySecret123 node license-key-gen.js 1y
```

На экран выводится один ключ. Его передают пользователю; пользователь вводит ключ в приложении (в левой панели — «Ввести ключ подписки»).

### Поведение в приложении

- В левой панели внизу отображается статус подписки: **«Подписка до ДД.ММ.ГГГГ (осталось N дн.)»** или ссылка **«Ввести ключ подписки»**.
- Если до конца подписки 7 дней или меньше — остаток дней выделяется предупреждением.
- Если подписка истекла — показывается сообщение и блокируется редактирование до ввода нового ключа.
- Ключ проверяется на сервере (подпись HMAC). Подделка ключа без знания секрета невозможна.

## Рекомендации при передаче пробной версии

### 1. Распространяйте сборку, а не исходники

- Соберите версию для распространения:  
  `npm run build`  
  В папке **dist/** будут обфусцированные `main.js` и `auth.js` и скопированные HTML/CSS.
- Для «пробной» передачи отдавайте папку **dist/** плюс серверную часть (например, `server-api.js`, `database.js`, `package.json`, папка `data/` при необходимости), **без** полного репозитория с исходным `main.js`/`auth.js`.
- Исходный код храните только у себя (и, при необходимости, в приватном репозитории).

### 2. В комплект поставки обязательно включайте

- Файл **LICENSE** — полный текст лицензии.
- Файл **README.md** (или краткую «Инструкцию») с указанием:  
  «© 2024–2026 Network-map. Все права защищены. Условия использования — в файле LICENSE.»

Так получатель явно видит, что это пробная/ограниченная версия и что права остаются за вами.

### 3. Письменное соглашение (по желанию)

При передаче организации или частному лицу можно оформить короткое письмо/соглашение, например:

- «Предоставляется пробная (демонстрационная) версия ПО «Карта локальной сети» на срок … для ознакомления. Использование только в указанных целях. Запрещены копирование, распространение и коммерческое использование без письменного согласия правообладателя.»

Подпись и дата фиксируют факт передачи на определённых условиях.
