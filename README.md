# Карта оптической сети

Веб-приложение для визуализации и управления сетевой инфраструктурой на базе Яндекс.Карт. Одна карта на всех: данные и синхронизация через единый сервер.

**Версия:** 1.1.8

## Возможности

- **Визуализация** — узлы, оптические кроссы, кабельные муфты, опоры связи, кабели и объекты GPON (OLT, сплиттер, ONU) на карте
- **Трассировка жил** — путь оптического волокна от узла до узла через муфты и кроссы; трассировка от порта OLT (в т.ч. все пути при разветвлении через сплиттер)
- **Редактирование** — добавление и удаление объектов, прокладка кабелей, соединение жил в кроссах и муфтах, подключение жил кросса к узлам сети
- **Кабели** — ВОЛС 4/8/16/24 жилы: от муфты/кросса до муфты/кросса с опорами как промежуточными точками
- **Кросс** — соединение жил между кабелями, вывод жил на узлы сети (выбор жилы и кнопка «Подключить к узлу»)
- **Группы** — объединение нескольких кроссов или узлов в одной точке
- **Поиск** — по названию в шапке, центрирование на объекте
- **Отмена и повтор** — до 20 действий за сессию: **Ctrl+Z** (отменить), **Ctrl+Y** (повторить); кнопки в шапке. Учитываются только ваши действия в текущей вкладке.
- **Выделение областью** — зажмите правую кнопку мыши и потяните по карте; после отпускания показывается панель с количеством объектов в области и кнопка «Удалить выделенное» (в режиме редактирования)
- **Фильтр карты** — показ только выбранных типов (узлы, кроссы, муфты, опоры, крепление, OLT, сплиттеры, ONU); кабели отображаются только между видимыми объектами
- **GPON** — OLT (приход от кросса, PON-порты, назначение жил), сплиттер (1:N, вход/выходы), ONU; подключение жилы к ONU и сплиттеру из кросса/муфты с прокладкой маршрута через опоры и крепления; выходы сплиттера на ONU/сплиттер с маршрутом; трассировка от порта OLT, в т.ч. все пути при сплиттере; экспорт/импорт полей GPON
- **Экспорт / Импорт** — JSON
- **Авторизация** — администратор (полный доступ) и пользователь (только просмотр)
- **История изменений** — журнал действий с фильтрами по пользователю, типу действия и датам; бейдж непрочитанных в шапке
- **Типы узлов** — настраиваемый тип узла сети (обычный / агрегации) с разными цветами и отображением цветов внутри группы узлов
- **Начальная позиция карты** — каждый пользователь может сохранить текущий вид как начальный (при открытии карта открывается в «своём» городе/районе)
- **Интерфейс карты** — складывающаяся левая панель с кнопкой-стрелкой; при скрытии панель освобождает место под карту
- **Тема** — светлая и тёмная (интерфейс и подложка карты)
- **Синхронизация** — совместная работа в реальном времени (пооперационная, как Эсборд), курсоры участников
- **Хранение** — карта, пользователи, история, настройки на сервере (файл `data/store.json`)
- **Резервные копии** — автоматическое ежедневное сохранение в `data/backups/` с датой в имени файла; хранятся 30 дней, старые удаляются

## Быстрый старт

1. Установите зависимости: `npm install`
2. Запустите сервер: `npm run api` (порт 3000 по умолчанию)
3. Откройте в браузере **http://localhost:3000**
4. Войдите (по умолчанию: `admin` / `admin123`)
5. В блоке **Синхронизация** нажмите **Подключиться** — карта загрузится и будет синхронизироваться с сервером
6. Переключитесь в режим **Редактирование** для добавления объектов и прокладки кабелей

Без сервера приложение не работает: при открытии показывается предложение подключиться к серверу.

## Режимы и объекты

| Режим | Описание |
|-------|----------|
| **Просмотр** | Навигация, клик по объектам — информация и трассировка. Редактирование недоступно. |
| **Редактирование** | Добавление/удаление объектов, прокладка кабелей, соединение жил. Только для пользователей с правами редактора. |

| Тип объекта | Назначение |
|-------------|------------|
| Узел сети | Конечное устройство; подключаются жилы из кроссов (в кроссе — «Подключить к узлу») |
| Оптический кросс | Соединение жил между кабелями, вывод жил на узлы, ONU и сплиттеры с прокладкой маршрута |
| Кабельная муфта | Сращивание жил между кабелями, вывод жил на ONU и сплиттеры с прокладкой маршрута |
| Опора связи | Транзитная точка (воздушные линии) |
| OLT (GPON) | Станция PON: приход от кросса, PON-порты, трассировка от порта (все пути при сплиттере) |
| Сплиттер | Делитель 1:N; входная жила (с маршрутом) и выходы на ONU/сплиттер (с маршрутом через опоры); трасса через сплиттер |
| ONU | Абонентский терминал; подключается жилой из кросса/муфты с прокладкой маршрута через опоры/крепления |
| Кабель | **ВОЛС** 4/8/16/24 жилы: от муфты/кросса/OLT до муфты/кросса/OLT, опоры и крепления — только промежуточные точки. Сплиттер и ONU не используются для прокладки кабеля ВОЛС. |

**Добавление:** в режиме редактирования выберите тип в боковой панели и кликните по карте.

**Прокладка кабеля:** **Проложить кабель** → выберите тип (4/8/16/24 жилы) → клик по начальной и конечной точке (муфта/кросс/OLT), при необходимости опоры и крепления как промежуточные точки. Начало и конец — не сплиттер, не ONU, не опора, не крепление.

**Кросс:** в баллуне кросса отображаются жилы кабелей; для свободной жилы доступна «Подключить к узлу» (выбор из списка). Линии кросс–узел отображаются пунктиром на карте.

**Выделение областью:** зажмите правую кнопку мыши и потяните по карте — появится прямоугольная область. После отпускания внизу отобразится панель со статистикой объектов (кроссы, узлы, муфты, опоры, кабели и т.д.). В режиме редактирования — кнопка «Удалить выделенное».

В карточке объекта доступны **Удалить** и **Дублировать** (в режиме редактирования), в т.ч. для OLT, сплиттера и ONU (копируются имя и параметры, приход/порты/вход-выходы сбрасываются). При удалении опоры, через которую проложен кабель, показывается предупреждение; кабель удаляется вместе с опорой.

## Горячие клавиши

| Клавиша | Действие |
|---------|----------|
| `Ctrl+Z` | Отменить последнее действие (до 20 шагов за сессию) |

## Синхронизация

- Сервер: тот же процесс, что и REST API (`npm run api`). WebSocket: путь `/sync`, по умолчанию тот же хост (например `ws://localhost:3000/sync`).
- При подключении клиент получает текущее состояние карты; дальнейшие изменения передаются операциями (без перезагрузки всей карты).
- В блоке синхронизации отображается список «В сети»; на карте видны курсоры других участников.

Отдельный старый сервер только синхронизации: `npm run sync` (server.js) — для сценариев без REST API.

## Хранение данных

- Всё хранится на сервере в `data/store.json` (карта, пользователи, сессии, история, настройки).
- Запускайте один экземпляр сервера (`npm run api`).
- **Резервные копии:** сервер раз в сутки сохраняет полную копию в `data/backups/backup-YYYY-MM-DD.json`. Хранятся последние 30 дней, более старые удаляются автоматически.

#### Работа с бэкапами

1. **Посмотреть список бэкапов** — в папке `data/backups/` лежат файлы `backup-2026-02-18.json`, `backup-2026-02-19.json` и т.д. (по одному на день).
2. **Восстановить данные из бэкапа:**
   - **Из приложения (рекомендуется):** войдите как администратор → в боковой панели откройте раздел **«Резервные копии»** → нажмите «Обновить список» → выберите нужную дату и нажмите **«Восстановить»** → подтвердите → после успешного восстановления обновите страницу (F5).
   - **Вручную:** остановите сервер (Ctrl+C) → при желании сохраните текущую базу (`data/store.json`) → скопируйте нужный файл бэкапа в `data/store.json` (например: `copy data\backups\backup-2026-02-18.json data\store.json`) → запустите сервер снова → обновите страницу в браузере.
3. **Ручная страховка** — кроме автоматических бэкапов можно в любой момент использовать **Экспорт карты** в приложении (Файлы → Экспорт карты) или вручную скопировать `data/store.json` в безопасное место.

### Настройка сервера

Скопируйте `server-config.example.json` в `server-config.json`. Параметры:

| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| `port` | 3000 | Порт |
| `host` | "0.0.0.0" | `0.0.0.0` — доступ по localhost и по IP в сети |

Переменные окружения `PORT` и `HOST` переопределяют файл.

## Технологии

- Яндекс.Карты API 2.1
- JavaScript (без фреймворков)
