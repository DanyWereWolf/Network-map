# Карта оптической сети

Программа для работы с оптической сетью — веб-сервис для визуализации и учёта волоконно-оптической и смежной инфраструктуры на базе Яндекс.Карт: ВОЛС, узлы с коммутаторами, оптические кроссы и муфты, GPON (OLT, сплиттер, ONU), камеры и медиаконвертеры с медными линиями к коммутатору. Одна карта на организацию: данные изолированы по компаниям, хранятся на наших серверах, совместная работа через синхронизацию в реальном времени.

**Версия:** 1.2.0

**Работа в облаке:** [volsmap.ru](https://volsmap.ru/) — лендинг и вход; карта после авторизации. Установка ПО у заказчика не требуется.

Ниже — инструкции для **локальной разработки** и администрирования своего экземпляра сервера (исходный код в репозитории).

## Возможности

- **Визуализация** — узлы (в т.ч. с коммутаторами), отдельные коммутаторы на карте, оптические кроссы, кабельные муфты, опоры связи, крепления, кабели ВОЛС, объекты GPON (OLT, сплиттер, ONU), камеры и медиаконвертеры
- **Кросс и муфта** — рабочее место жил: вкладки «Схема» и «Таблица», соединения, вывод на узлы/ONU/сплиттеры; у кросса — учёт медных портов (RJ45)
- **Медные линии** — подключение камеры или медиаконвертера к порту коммутатора узла или отдельного коммутатора; учёт портов RJ45 и SFP
- **Трассировка жил** — путь оптического волокна от узла до узла через муфты и кроссы; от порта OLT (в т.ч. все пути при разветвлении через сплиттер); из карточки камеры — по жилам кросса, подключённым к узлу коммутатора (или по входной жиле медиаконвертера)
- **Редактирование** — добавление и удаление объектов, прокладка кабелей ВОЛС и медных линий, соединение жил в кроссах и муфтах, подключение жил кросса к узлам сети
- **Кабели** — ВОЛС 4/8/16/24 жилы: от муфты/кросса до муфты/кросса с опорами как промежуточными точками; медные — от коммутатора к камере, медиаконвертеру и др. по правилам карточки объекта
- **Кросс** — соединение жил между кабелями, вывод жил на узлы сети (выбор жилы и кнопка «Подключить к узлу»)
- **Группы** — объединение нескольких кроссов или узлов в одной точке
- **Поиск** — по названию в шапке, центрирование на объекте
- **Отмена и повтор** — до 20 действий за сессию: **Ctrl+Z** (отменить), **Ctrl+Y** (повторить); кнопки в шапке. Учитываются только ваши действия в текущей вкладке
- **Выделение областью** — зажмите правую кнопку мыши и потяните по карте; после отпускания панель со статистикой объектов и кнопка «Удалить выделенное» (в режиме редактирования)
- **Фильтр карты** — показ только выбранных типов (узлы, в т.ч. «только узлы агрегации», кроссы, муфты, опоры, крепление, OLT, сплиттеры, ONU, камеры, медиаконвертеры); кабели отображаются только между видимыми объектами
- **GPON** — OLT (приход от кросса, PON-порты, назначение жил), сплиттер (1:N, вход/выходы), ONU; подключение жилы к ONU и сплиттеру из кросса/муфты с прокладкой маршрута через опоры и крепления; выходы сплиттера на ONU/сплиттер с маршрутом; трассировка от порта OLT, в т.ч. все пути при сплиттере; экспорт/импорт полей GPON
- **Справочник устройств** — производители и модели для коммутаторов, OLT, ONU, камер и медиаконвертеров (вкладки в модальном окне администратора)
- **Экспорт / Импорт** — JSON карты организации
- **Авторизация** — администратор организации (редактирование) и пользователь (просмотр); глобальный администратор — панель `site-admin.html`
- **Организации и подписка** — отдельная карта, история и бэкапы на организацию; тарифы с лимитом одновременных пользователей; личный кабинет (тариф, срок подписки); пробный период 14 дней при регистрации
- **История изменений** — журнал действий с фильтрами по пользователю, типу действия и датам; бейдж непрочитанных в шапке
- **Типы узлов** — обычный / агрегации с разными цветами и отображением внутри группы узлов
- **Начальная позиция карты** — каждый пользователь может сохранить текущий вид как начальный
- **Мобильный просмотр** — на экранах шириной ≤768px доступен только режим просмотра; редактирование — с компьютера или планшета в альбомной ориентации с большей шириной
- **Интерфейс** — складывающаяся левая панель; светлая и тёмная тема (интерфейс и подложка карты)
- **Синхронизация** — совместная работа в реальном времени (пооперационная), курсоры участников
- **Хранение** — `data/store.json`: организации, пользователи, карты по организациям (`mapDataByOrg`), история, настройки, тарифы витрины
- **Резервные копии** — ежедневно для каждой организации в `data/backups/<id_организации>/backup-YYYY-MM-DD.json`; хранятся 30 дней

## Быстрый старт (для пользователей сервиса)

1. Откройте [volsmap.ru](https://volsmap.ru/) и нажмите **Попробовать 14 дней** или **Войти**
2. Зарегистрируйте организацию или войдите под выданными учётными данными
3. В приложении откройте блок **Синхронизация** → **Подключиться** — загрузится карта вашей организации
4. На широком экране включите **Редактирование** для добавления объектов и кабелей

Данные хранятся на серверах сервиса; отдельная установка у вас не нужна.

## Локальная разработка

1. `npm install`
2. `server-config.example.json` → `server-config.json`, укажите `yandexMapsApiKey`
3. `npm run api` (порт 3000)
4. **http://localhost:3000** — лендинг; карта — `/index.html`
5. Вход (часто `admin` / `admin123` при первом запуске) → **Синхронизация** → **Подключиться**

Без запущенного API карта не работает. Без ключа Яндекс.Карт подложка не загрузится.

## Режимы и объекты

| Режим | Описание |
|-------|----------|
| **Просмотр** | Навигация, клик по объектам — информация и трассировка. Редактирование недоступно. На узком экране (≤768px) — единственный доступный режим. |
| **Редактирование** | Добавление/удаление объектов, прокладка кабелей, соединение жил. Только для пользователей с правами редактора и ширины окна >768px. |

| Тип объекта | Назначение |
|-------------|------------|
| Узел сети | Конечное устройство на карте; в карточке — коммутаторы (RJ45 и SFP), медные линии к камере или медиаконвертеру; жилы ВОЛС с кросса («Подключить к узлу») |
| Оптический кросс | Соединение жил (схема/таблица), медные порты, вывод жил на узлы, ONU и сплиттеры с маршрутом |
| Кабельная муфта | Сращивание жил (схема/таблица), вывод жил на ONU и сплиттеры с маршрутом |
| Опора связи | Транзитная точка (воздушные линии) |
| OLT (GPON) | Станция PON: приход от кросса, PON-порты, трассировка от порта (все пути при сплиттере) |
| Сплиттер | Делитель 1:N; входная жила (с маршрутом) и выходы на ONU/сплиттер (с маршрутом через опоры) |
| ONU | Абонентский терминал; подключается жилой из кросса/муфты с маршрутом |
| Камера | Только медный кабель от коммутатора; в карточке — трассировка оптики к узлу |
| Медиаконвертер | Оптика с муфты/кросса, медь к коммутатору или камере |
| Коммутатор (отдельно на карте) | Точка с привязкой к родительскому узлу; медные и SFP-порты |
| Кабель | **ВОЛС** 4/8/16/24 жилы; **медный** — от коммутатора к камере, МК и т.д.; опоры и крепления — промежуточные точки для ВОЛС |

**Добавление:** в режиме редактирования выберите тип в боковой панели и кликните по карте.

**Прокладка кабеля ВОЛС:** **Проложить кабель** → тип жил → начальная и конечная точка (муфта/кросс/OLT), при необходимости опоры и крепления. **Медный кабель** — из карточки узла, коммутатора или медиаконвертера.

**Кросс / муфта:** карточка объекта открывает рабочее место жил (схема соединений и таблица). Для свободной жилы — «Подключить к узлу», к ONU, к сплиттеру.

**Выделение областью:** ПКМ + протянуть — прямоугольник; статистика и «Удалить выделенное» в режиме редактирования.

В карточке объекта: **Удалить**, **Дублировать**, **Сохранить** (в рабочем месте кросса/муфты). При удалении опоры с кабелем — предупреждение.

## Горячие клавиши

| Клавиша | Действие |
|---------|----------|
| `Ctrl+Z` | Отменить последнее действие (до 20 шагов за сессию) |
| `Ctrl+Y` | Повторить отменённое действие |
| `Escape` | Отмена текущей операции (размещение, кабель, жила) |

## Синхронизация

- Сервер: тот же процесс, что и REST API (`npm run api`). WebSocket: `/sync`, по умолчанию тот же хост (`ws://localhost:3000/sync`).
- При подключении клиент получает состояние карты своей организации; изменения передаются операциями.
- Список «В сети» и курсоры других участников на карте.

Отдельный сервер только синхронизации: `npm run sync` (`server.js`) — без REST API.

## Хранение данных

**На сервисе (volsmap.ru):** карты, пользователи, история и бэкапы хранятся на наших серверах; у каждой организации изолированный набор данных.

**В локальной копии для разработки:** единый файл `data/store.json` (`mapDataByOrg`, `historyByOrg`, `settingsByOrg`, тарифы). Один процесс `npm run api` на машине разработчика.

- **Резервные копии:** раз в сутки для каждой организации — `data/backups/<id_организации>/backup-YYYY-MM-DD.json`, 30 последних дней (на сервере сервиса — в том же каталоге на диске хоста).

#### Работа с бэкапами

1. **Список** — в `data/backups/<id_организации>/` по одному файлу на день.
2. **Восстановление из приложения:** администратор организации → **Резервные копии** → дата → **Восстановить** → обновить страницу (F5). Восстанавливаются только данные этой организации.
3. **Вручную:** остановить сервер → скопировать нужный бэкап, при необходимости вручную подставить `mapData` / `history` / `settings` в `store.json` для ключа организации в `mapDataByOrg` и т.д. → запустить сервер.
4. **Экспорт карты** (Файлы → Экспорт) — дополнительная ручная страховка.

### Настройка сервера

Скопируйте `server-config.example.json` в `server-config.json`:

| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| `port` | 3000 | Порт HTTP |
| `host` | `0.0.0.0` | Доступ по localhost и по IP в сети |
| `yandexMapsApiKey` | — | Ключ API Яндекс.Карт (обязателен для карты) |
| `publicSiteUrl` | — | Базовый URL сайта (например `https://volsmap.ru`) для ссылок и SEO |

Переменные окружения `PORT` и `HOST` переопределяют порт и хост.

## Технологии

- Яндекс.Карты API 2.1 (ключ с сервера `/api/public-config`)
- JavaScript без фреймворков на клиенте
- CSS-переменные, тёмная подложка карты
- Сервер: Express, WebSocket (`ws`), JSON-хранилище
