# План работ по GPON

Краткий аудит и план доработок по объектам GPON (OLT, сплиттер, ONU) в карте оптической сети.

---

## 1. Что уже сделано

- **Типы объектов:** OLT, сплиттер, ONU — добавлены в форму «Добавить объект», на карту ставятся по клику.
- **OLT:**
  - Один приход от кросса (жила, не порт); настройка «приход от кросса» в модалке OLT.
  - PON-порты и таблица назначений: какая жила (кабель + номер жилы) подключена к какому порту.
  - На противоположном конце жилы (кросс/муфта) записывается `oltConnections` — в таблице жил отображается «OLT &lt;имя&gt;, порт N».
  - Трассировка от порта OLT: кнопка «Трассировка» по порту → показ пути от OLT до конечной точки (кросс/муфта/ONU).
- **Кросс/муфта:** хранение и отображение `oltConnections`, `onuConnections`; трасса учитывает вывод на ONU.
- **Сплиттер:** тип объекта, поля `inputFiber`, `outputConnections`, `splitRatio`; сохранение/загрузка; конечная точка кабеля.
- **ONU:** тип объекта, отображение в трассе при `onuConnections` в кроссах.
- **Кабели:** конечными точками могут быть OLT, сплиттер, ONU (вместе с кроссом, муфтой, креплением).
- **Фильтр карты:** добавлены OLT, сплиттер, ONU в состояние фильтра и чекбоксы в блок «Фильтр карты»; объекты GPON можно включать/выключать, кабели к ним считаются видимыми по тем же правилам.
- **Сохранение/экспорт/импорт:** все поля GPON сериализуются и восстанавливаются.

Трассировка: при заходе на сплиттер по входной жиле — продолжение по первому выходу; при заходе по выходной — возврат по входной жиле к кроссу/муфте. Добавлена `traceAllFiberPathsFromObject`: при разветвлении собираются все пути по выходам сплиттера; трасса от порта OLT при нескольких путях показывает «Найдено путей: N» и путь 1. В трассе отображается сплиттер (иконка 🔀).

4. **Дублирование OLT / сплиттера / ONU** — **сделано.** В `duplicateObject` дублирование разрешено: создаётся копия объекта со сдвигом координат, имя «… (копия)». После дублирования вызывается `saveData()`.

## 5. Пересмотр системы GPON (аудит)

### 5.1 Исправлено при пересмотре

- **Удаление OLT:** при удалении OLT во всех кроссах и муфтах очищаются записи в `oltConnections`, ссылающиеся на этот OLT (`oltId === uniqueId`). После удаления вызывается `updateAllConnectionLines()`.
- **Удаление ONU:** при удалении ONU во всех кроссах и муфтах очищаются записи в `onuConnections`, ссылающиеся на эту ONU. Линии кросс/муфта–ONU перестраиваются через `updateAllConnectionLines()`.
- **Удаление кабеля:** при удалении кабеля по его `uniqueId`:
  - в каждом кросс/муфте из `oltConnections` и `onuConnections` удаляются все ключи вида `cableId-fiberNumber` для этого кабеля;
  - в каждом OLT из `portAssignments` удаляются назначения портов с `cableId === cableUniqueId`;
  - после удаления вызывается `updateAllConnectionLines()`.

### 5.2 Прокладка кабеля (проверка логики)

- **Клики по объектам:** при прокладке кабеля начальная и конечная точки могут быть cross, sleeve, support, attachment, **olt, splitter, onu** — учтено в `cableEndpoints` в обработчике клика по карте.
- **Клик в пустое место (автовыбор):** при выборе ближайшего объекта как конечной точки раньше использовался список без GPON. **Исправлено:** `validCableEndpoints` при автовыборе дополнен `'olt', 'splitter', 'onu'`, чтобы кабель можно было довести до OLT/сплиттера/ONU кликом рядом с объектом.
- **Сообщения об ошибках:** обновлены формулировки («муфта, кросс, крепление, OLT, сплиттер или ONU») для согласованности с допустимыми конечными точками.
- **Превью кабеля:** при движении мыши превью привязывается к объекту под курсором через `findObjectAtCoords` (исключены только cable/cableLabel), поэтому OLT/сплиттер/ONU корректно используются как цели привязки.
- **Создание кабеля:** `createCableFromPoints` проверяет типы через `validEndpoints` (включая olt, splitter, onu); ограничение по волокнам (`maxFibers`) применяется только к муфтам (sleeve). OLT/сплиттер/ONU не используют `usedFibers` при прокладке — назначение жил задаётся отдельно (порты OLT, вход/выходы сплиттера, подключение к ONU в кросс/муфте).

### 5.3 Проверено, изменений не требуется

- **Поиск:** `searchObjects` ищет по имени и типу; OLT, сплиттер, ONU имеют имя и тип — находятся корректно.
- **Конечные точки кабеля:** `cableEndpoints` / `validEndpoints` включают `olt`, `splitter`, `onu` во всех местах (размещение, прокладка кабеля).
- **Трассировка:** основной сценарий — от кросса/муфты (`traceFiberPathFromObject`) и от порта OLT (`showFiberTraceFromOLTPort`); учтены сплиттер, `onuConnections`, `oltConnections`. Функция `showFiberTrace(cableId, fiberNumber)` (трасса от жилы кабеля) вызывается только из кода трассы по кабелю; при достижении OLT/ONU/сплиттера трасса останавливается (`else { break }`). Расширение этой ветки под ONU/сплиттер при необходимости можно добавить позже.
